# Narrative plan

## Audience

Course examiner and capstone discussion panel evaluating practical understanding
of persona design, tools, MCP, planning, RAG, memory, and Docker.

## Objective

Explain the operational problem, show an inspectable agent architecture, and
demonstrate that document evidence becomes sourced answers, risks, and owned
actions in a reliable no-key demo.

## Narrative arc

1. Clinical operations has a document-to-action problem.
2. The solution is an agent workflow with separate knowledge, memory, tools,
   and standardized MCP exposure.
3. The live demo proves the workflow and maps it directly to course concepts.

## Slide list

1. **Problem & Solution** — concise problem statement, solution statement, and
   four capability cards.
2. **System Architecture** — left-to-right architecture flow with short layer
   descriptions and a Docker deployment foundation.
3. **Demo Flow & Course Mapping** — numbered demo sequence on the left and seven
   course-concept chips on the right.

## Source plan

All content comes from the local project implementation and documentation:
`app/`, `README.md`, `docs/architecture.md`, and the verified demo workflow. No
external facts or citations are required.

## Visual system

- 16:9 light canvas with navy text, blue/teal accents, and restrained amber.
- Aptos Display/Aptos typography for reliable PowerPoint compatibility.
- Rounded cards, simple line connectors, small numbered markers, and generous
  whitespace.
- Consistent top title band, bottom progress line, and small capstone label.

## Image and icon plan

No generated imagery is needed. The subject is a system workflow, so editable
native cards, connectors, document/database/tool symbols, and architecture
shapes communicate the content more clearly than decorative raster art.

## Asset needs

No external assets. All visible content is editable PowerPoint text and native
shapes.

## Editability plan

Titles, bullets, labels, cards, architecture nodes, arrows, and speaker notes
remain editable. The final deck contains exactly three slides and no
screenshot-only content.
